# Git Expert Skill

A specialized skill for seamless Git synchronization between the Arena AI Workspace and the user's local VS Code environment via GitHub.

## 1. Context Gathering
*   **Repository State**: Check if the current workspace is a Git repository (`ls -a` for `.git`).
*   **Identity Check**: Verify if `user.email` and `user.name` are configured.
*   **Authentication**: Access the stored Personal Access Token (PAT) securely for remote calls.

## 2. Execution Workflow

### Step 1: Commit Analysis
*   Run `git diff --stat` to identify modified files.
*   Generate a **Conventional Commit** message:
    *   `feat:` for new features or skills.
    *   `fix:` for bug fixes.
    *   `docs:` for documentation changes.
    *   `refactor:` for code changes that neither fix a bug nor add a feature.

### Step 2: Branch Management
*   Ensure the agent is on the correct branch (default: `main`).
*   Create new branches using `git checkout -b <name>` if a feature branch is requested.

### Step 3: Synchronization (The "Sync" Pulse)
1.  **Pull First**: Always execute `git pull origin <branch>` to incorporate remote changes and prevent conflicts.
2.  **Add & Commit**: Stage all changes and commit with the generated message.
3.  **Push**: Execute `git push origin <branch>` to update the remote repository.

## 3. Output Format
After each sync, provide a summary:
*   ✅ **Status**: Success/Failure.
*   📝 **Commit Message**: The description used for the update.
*   📂 **Files Changed**: List of key files modified.
*   🚀 **Next Step**: Instructions for the user (e.g., "Run `git pull` in your VS Code terminal").

## 4. Safety Boundaries
*   **Credential Protection**: Never log the raw GitHub Token or include it in plain text files.
*   **Destructive Actions**: Ask for explicit confirmation before using `--force` or deleting branches.
*   **Conflict Resolution**: If a merge conflict occurs, pause and ask the user for guidance before proceeding.
